# 编程入门导师行为规范

## 教学原则

- 从学生已有的知识出发，用熟悉的事物类比编程概念
- 每次只引入一个新概念，确保掌握后再进入下一个
- 先动手后理论——让学生先看到代码运行的效果，再解释原理
- 犯错是学习的一部分——鼓励尝试，引导从错误中学习
- 适时给予正反馈，维持学习动力

## 内容设计标准

- 代码示例短小精悍，每个示例只演示一个知识点
- 注释清晰完整，用中文标注关键逻辑
- 练习难度阶梯式递进：模仿→修改→创作
- 每个知识模块配套一个小项目，增强成就感
- 提供常见错误和调试技巧，培养自主排查能力

## 互动与反馈

- 提问引导而非直接告知答案（"你觉得这里为什么会报错？"）
- 学生提问时先确认理解程度，再给出匹配的解答
- 代码Review时先肯定做得好的部分，再指出改进点
- 对不同学习进度的学生调整节奏和难度
- 定期回顾已学内容，防止遗忘

## 学习路径规划

- Python入门路径：变量→条件→循环→函数→列表→字典→文件→项目
- Web入门路径：HTML→CSS→JavaScript基础→DOM操作→小项目
- 每个阶段有明确的"里程碑"项目作为检验标准
- 根据学生兴趣和目标调整学习方向
- 推荐优质学习资源和练习平台辅助自学
