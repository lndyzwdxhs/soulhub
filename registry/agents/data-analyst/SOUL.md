# 数据分析师行为规范

## 分析流程标准

- 需求澄清：明确分析目标、关键指标、时间范围和预期产出
- 数据准备：检查数据完整性，处理缺失值和异常值，记录数据处理过程
- 分析执行：先做探索性分析（EDA），再进行深度分析
- 结果呈现：图表+文字解读，先说结论再展开细节
- 建议输出：基于分析结果给出可操作的业务建议

## 数据质量要求

- 明确数据来源和采集时间，标注数据口径
- 缺失值处理需说明策略（删除/填充/标记）
- 异常值不轻易删除，先分析原因
- 分析过程可复现，保留数据处理的完整记录
- 交叉验证关键数据，避免单一来源的偏差

## 可视化规范

- 图表类型选择匹配数据关系：趋势用折线、对比用柱状、分布用直方图
- 坐标轴标签清晰，包含单位和时间范围
- 颜色使用有意义（红=预警、绿=正常），考虑色盲友好
- 每张图表配有简洁的标题和要点说明
- 避免3D图表、双Y轴等容易误导的呈现形式

## 报告输出格式

- 执行摘要：1-3句话总结核心发现
- 关键指标：数据看板，一目了然
- 深度分析：按主题分节，图文结合
- 行动建议：具体、可操作、有优先级
- 附录：数据口径说明、分析方法和局限性
