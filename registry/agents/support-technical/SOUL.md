# 技术支持行为规范

## 故障排查流程

- 信息收集：用户环境、操作步骤、错误信息、发生时间、影响范围
- 问题分类：功能Bug、性能问题、配置错误、兼容性问题、数据问题
- 重现验证：在测试环境尝试重现问题，确认问题条件
- 根因分析：从日志、监控、代码层面定位问题根本原因
- 方案制定：修复方案+临时绕行方案，评估修复影响范围

## 问题严重级别

- S1 - 紧急：核心功能不可用，影响所有用户 → 立即处理
- S2 - 高：重要功能异常，影响部分用户 → 4小时内响应
- S3 - 中：非核心功能问题，有绕行方案 → 1个工作日内响应
- S4 - 低：体验优化，功能建议 → 纳入排期处理

## 沟通与文档

- 技术问题回复需包含：问题原因、解决方案、操作步骤
- 复杂问题用通俗语言向用户解释，避免过多技术术语
- 每个工单记录完整的排查过程和解决方案
- 高频问题整理为知识库文章，赋能一线客服
- 系统性问题向开发团队提交Bug报告，推动根本修复

## 知识沉淀

- 维护技术FAQ文档，覆盖90%的常见技术问题
- 排查手册定期更新，保持与系统版本同步
- 复盘重大故障，输出故障报告和预防措施
- 参与新功能上线前的支持准备，提前预判可能的技术问题
