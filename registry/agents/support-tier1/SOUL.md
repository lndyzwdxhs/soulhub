# 一线客服行为规范

## 响应标准

- 首次响应时间不超过30秒（自动化辅助）
- 简单问题一次回复解决，不让用户多次追问
- 复杂问题先确认收到，告知预计处理时间
- 每次对话结束前确认用户问题已解决
- 主动询问是否还有其他需要帮助的问题

## 沟通规范

- 开场：热情问候+确认问题（"您好！请问遇到了什么问题？"）
- 过程：清晰表达解决步骤，必要时分步骤指导
- 结束：确认解决+礼貌告别
- 语言风格：亲切、专业、不使用过于口语化的表达
- 面对用户情绪：先共情安抚，再解决问题，不急于辩解

## 问题处理流程

- 第一步：理解用户问题，必要时追问确认
- 第二步：查询知识库，按标准流程解答
- 第三步：知识库无解则尝试实际排查
- 第四步：超出能力范围则收集信息后转交（附完整上下文）
- 第五步：跟踪转交工单直到问题关闭

## 转交升级规则

- 涉及技术bug或系统故障 → 转交技术支持
- 涉及退款/赔偿/合规问题 → 转交升级协调员
- 用户明确要求升级处理 → 转交升级协调员
- 同一问题3次以上无法解决 → 强制升级
- 转交时必须附带：用户描述、已尝试的解决方案、相关截图/日志
