# Python专家行为规范

## 编码标准

- 遵循PEP 8代码风格规范，使用black格式化
- 所有函数和类必须添加类型注解（typing模块）
- 编写Google风格的docstring，包含参数说明和返回值
- 变量和函数命名清晰自解释，避免单字母变量（循环索引除外）
- 使用pathlib处理文件路径，避免字符串拼接

## 工程实践

- 项目结构遵循Python最佳实践：src布局、pyproject.toml管理依赖
- 使用虚拟环境隔离依赖，明确指定依赖版本
- 核心逻辑必须有单元测试，覆盖率目标80%+
- 异常处理要精确，不使用裸except，记录有意义的错误信息
- 敏感信息使用环境变量，不硬编码在代码中

## 性能与安全

- 大数据处理优先使用生成器，避免一次性加载全部数据到内存
- I/O密集型任务考虑使用asyncio或多线程
- CPU密集型任务考虑使用multiprocessing
- 外部输入必须进行校验和清理，防止注入攻击
- 数据库操作使用参数化查询，不拼接SQL

## 交付标准

- 代码交付必须包含：源代码、测试、依赖文件、README
- 提供清晰的运行说明和环境配置步骤
- 关键设计决策需在代码注释或文档中说明理由
- 复杂逻辑附带使用示例（example/或doctest）
